# 07 — Pipeline: «Interactive AI-Clip Stage + reusable plugins»

> **Уточнённая граница (по итогам сверки).** Ограничение заказчика «без ручной / без сложной процедурной анимации» НЕ значит «запретить физику и частицы». Оно значит: **запретить bespoke-код под каждую карту, ручную/кадровую organic-анимацию и сложные симуляции без reuse.** Физика и FX входят в пайплайн как **переиспользуемые config-driven плагины, собранные ОДИН раз** (Codex/Claude Code это умеют); на карту идёт только **JSON-конфиг + тюнинг**.
> **Что это:** базовый clip-движок + небольшой набор reusable code-плагинов. Развивает POC (`poc-dino-companion`). Частично реабилитирует `05` §2 и возвращает physics-механики `02` (как плагины).

---

## 0. Правильная граница (что запрещаем на самом деле)

| | |
|---|---|
| **НЕ запрещаем** | физику (Arcade/Matter), частицы, простые процедурные эффекты, tweens, камеру — **как reusable config-driven плагины** |
| **❌ ЗАПРЕЩАЕМ** | (1) bespoke per-card TypeScript; (2) ручную/покадровую **organic-анимацию** персонажа; (3) сложные симуляции (fluid/cloth/ragdoll) **без многократного reuse**; (4) ручной per-card тюнинг **нового кода** |

**Кто что делает:**
- **AI-видео / sprite-atlas** → жизнь органического персонажа и эмоциональные моменты (характерное движение T-Rex, актёрский эпизод).
- **Reusable code-плагины** → настоящая игра (физика, FX, контроль): пишутся раз, конфигурируются данными.
- **AI (LLM)** → генерит ассеты и **per-card конфиги** (раскладки, силы, массы, пресеты, условия победы).
- **Человек** → принимает новый модуль **один раз** (Factory Rule §2) и проверяет итоговые карты (плейтест).

**Три источника движения (переформулировано):** (0) AI-baked клипы/спрайты — organic-жизнь; (1) reusable-плагины — физика/FX/трансформы по конфигу; (2) ввод рулит значением (drag/scrub). Запрещено только: **новый код и ручная анимация под отдельную карту.**

## 1. Архитектура: Stage + plugins

```
[ Card JSON ]  →  [ Interactive AI-Clip Stage ]  ← reusable plugins
                         │                            transform-tween
   AI-ассеты ────────────┘                            particle-fx
   (image, i2v clips, alpha-sprites, TTS, SFX)        arcade-physics
                                                      matter-slingshot
                                                      matter-stack
                                                      camera-fx
                                                      mask-reveal
```

**Базовый Stage** (как `06`-clip-движок): states · ветвления · AI-клипы · alpha-акторы · tap · drag-to-zone · hold · scrub · show/hide · audio.
**Reusable-плагины** (по одному разу, config-driven): см. список выше.
**Карта = JSON, ноль собственного TypeScript:**

```json
{
  "engine": "interactive-stage",
  "mechanic": "catch-dodge",
  "plugins": [
    { "type": "arcade-physics", "gravity": 300, "spawn": "hatchlings", "hazard": "rocks" },
    { "type": "particle-fx", "preset": "success-burst", "trigger": "catch" },
    { "type": "camera-fx", "preset": "shake", "trigger": "hazard-hit" }
  ],
  "assets": { "actor": "sprites/hatchling-loop.webm", "bg": "art/nest-scene.png" },
  "win": { "rule": "saved >= 8", "fail_outcome": "soft" },
  "pressure": "quest",
  "learning": { "fact": "детёныши держатся гнезда; камнепад — угроза", "win_requires_understanding": true }
}
```

**Движок: Phaser.** Arcade Physics (скорость/столкновения/гравитация) и Matter (формы/суставы/пружины/ограничения) **встроены** → минимум своего кода под Factory Rule. PixiJS — только если позже понадобится кастомный рендер-слой.

## 2. ⭐ Factory Rule — единственное правило, когда вообще можно писать новый код

Новый code-модуль разрешён, ТОЛЬКО если он одновременно:
1. **пригодится ≥ нескольким картам** (не bespoke под одну);
2. имеет **конфигурационную схему** (Zod) — карта параметризует его данными;
3. имеет **автотесты** (reachability, разрешимость, отсутствие тупиков);
4. **проверен на слабом устройстве** (перф, число одновременных физ-систем/видео-декодеров);
5. **fixed-timestep** (детерминизм — иначе рекорды не воспроизводимы);
6. после создания **используется AI без программиста** (только конфиг).

**Per-card после этого:** AI генерит конфиг; человек делает тюнинг + плейтест. **Нового кода на карту — ноль.**

> ⚠️ **Честность по стоимости (`05` §8):** reuse КОДА ≠ $0/карта. Per-card остаётся **authoring**: конфиг + тюнинг + плейтест. Дёшево там, где тюнится раз (Catch/Dodge, Slingshot с параметрической раскладкой); дороже на Balance/Stack/TD («заметный тюнинг каждого уровня»). Бюджетируем authoring-cost отдельно от engine-cost.

## 3. Палитра механик — RE-широкая (clip + physics плагины, один Stage)

**Clip-механики** (organic-жизнь = AI-клип; §5 промпты):
- `branching-choice` · `drag-to-zone` · `tap-target-live` · `drag-scrub` · `hold-release`.

**Physics-плагин-механики** (контроль/непредсказуемость = reusable-плагин):
- `catch-dodge` (Arcade) — лови нужное / уворачивайся; Slingshot (Matter) — оттяни-пусти; `balance-stack` (Matter) — равновесие/стопка; простое `destroy` (слои объекта + impact-FX); простая погоня (Arcade).

**Что даёт физика, чего клипы НЕ дают** (см. §8): немного непредсказуемый исход · мастерство · множество решений · реальный контроль · «ещё разок». Поэтому колода = **микс clip + physics**, не только ветки клипов.

## 4. Что включить в ПЕРВЫЙ пайплайн (4 уровня) + cost-приор

| Уровень | Модули | Per-card |
|---|---|---|
| **Обязательно** | clip-state-machine; tap/drag/hold/scrub; transform-tween; camera-fx; sprite-atlas; alpha-video; базовый particle-fx-пресет-набор | конфиг (≈$0) |
| **Почти обязательно** | **Arcade Physics** (скорость/столкновения/падение/погоня/Catch-Dodge) | конфиг + малый тюнинг |
| **Следующий reusable-модуль** | **Matter Slingshot**, **Matter Balance/Stack** | раскладка + тюнинг |
| **Отложить** | fluid / cloth / ragdoll / свободная сложная физика / per-card симуляция / procedural character-rig / Tower-Defense (до проверки базового engagement) | — |

Cost-приор (инженерная оценка, не статистика): tweens/камера/particle-пресеты — низкая первичная, ~нулевая на карту; Arcade — средняя первичная, конфиг+малый тюнинг; Slingshot/Stack — средняя, раскладка+тюнинг; TD/псевдожидкость — высокая (баланс/перф-QA), отложить; organic-character-анимация → **только AI-video→sprite**, не код.

## 5. Промпт-рецепты клипов под Kling (i2v) — для organic-персонажей и эмоций

Правила консистентности: **один input-кадр (арт карты) на ВСЕ ветви** (существо/стиль идентичны); короткие 3–5 с; **camera static/locked**; нейтральный/контрастный фон для альфа-актёров; `generate_audio:true` для рёва; первый кадр ветви ≈ последний кадр intro.
Шаблоны (динозавры): `intro` «T-Rex медленно поворачивает голову, фиксирует взгляд, рык, 4с» · `freeze_win` «нюхает воздух, не находит неподвижную добычу, уходит, 4с» · `run_caught` (мягко) «комичный near-miss, экран-шейк, без крови, 3с» · `idle-loop` актёр «дышит, моргает, дёргает хвостом, бесшовный луп, изолирован на тёмном фоне, 3с».
Негативы: `blurry, extra limbs, text, watermark, fast camera, scene cut, gore`.

## 6. Что это пере-скоупит в `05`/`06`

- **`05` §2 — ЧАСТИЧНО РЕАБИЛИТИРОВАН.** Компонуемые слои/модули поведения возвращаются, НО строго как **reusable config-driven плагины под Factory Rule** (§2) — не bespoke per-card движки. Процедурную physics-механику можно, **physics на карту руками кодить — нельзя**.
- **`02` library** — physics-механики (Catch/Dodge, Slingshot, Balance/Stack, simple Destroy) **возвращаются в палитру** как плагины (раньше я их вычеркнул под строгим прочтением).
- **`06`** обновлён: срез теперь = карта №1 (clip-branching) + карта №2 (**physics-плагин**), чтобы проверить и Stage, и плагин-архитектуру, и тезис «физика даёт replay, которого клипы не дают».
- Остаётся в силе: манифест/сигнатура (`05` §1), pressure-профили (§3), learning-gate (§4), actor-pack=AI-спрайт (§5), deck-packs (§6), операционно-безопасный слой (§7).

## 7. Почему НЕ только AI-клипы

Clip-only хорош для: выбор-и-последствие · реакция существа · кормление · stealth · reveal · сюжетные исходы. **Но** если вся серия — только ветки клипов, ребёнок быстро вскрывает структуру «ограниченный выбор → готовый ответ», и интерес падает. Физика даёт то, чего клипы не дают (§3). Причём простой Slingshot/Stack **после создания движка может быть дешевле**, чем генерировать и чистить по несколько outcome-видео на каждую карту.

**Курс:** AI-клипы создают персонажей и эмоции; reusable physics/FX-плагины создают настоящую игру; AI генерит ассеты и конфиги; человек принимает новый модуль один раз и проверяет итоговые карты. **Вычёркиваем только: ручную анимацию · ручной per-card код · bespoke на одну карту · сложные симуляции без reuse.**
