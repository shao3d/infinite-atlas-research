# Контекст для ревизора: что такое Infinite Atlas

*Депесонализированная сводка. Личные/финансовые детали участников опущены намеренно — они не нужны для оценки исследования.*

## Продукт
**Infinite Atlas** — фиджитал-платформа детского обучения для EU и US. Физические коллекционные карточные колоды (колода = 15–30 карт), которые «оживают» в бесплатном офлайн-PWA-приложении. Семья покупает карты в обычной рознице, ребёнок выбирает/сканирует карту → открывается короткий интерактив (30–120 сек) на мобильном. Без интернета, без аккаунта.

**Аудитория:** дети 7–12 лет.
**Темы колод (кандидаты):** космос, динозавры, древний Рим/Греция, океан, роботы / «как всё устроено», человеческое тело, история.

## Бизнес-модель
Успех определяется двумя воротами:
1. **Полка в рознице** (заметность, дистрибуция — Ашан/супермаркеты/Amazon).
2. **Повторные покупки** — «папа, купи следующую колоду!».

Цифровой компаньон **бесплатный**; его задача — не монетизация, а **вовлечение и запоминаемость**, чтобы РЕБЁНОК стал инициатором покупки. Базовый принцип — *«развлекая обучай»*, без школьно-справочного занудства. Ключевая ставка: себестоимость продакшена упала в ~100× за счёт AI, и можно выпускать тысячи колод на разных языках/темах быстрее, чем традиционные детские редакции.

## Главное напряжение (что исследовали)
Баланс между:
- **ВОВЛЕЧЕНИЕМ** (эмоции, кураж, «вау», желание вернуться и просить новую колоду), и
- **СТОИМОСТЬЮ ПРОДАКШЕНА** интерактивов через AI-конвейер.

Главный рычаг экономии — **переиспользование одного окружения/движка на много карт**. Эталон — космос (чернота + шарики планет: одно окружение закрывает 15+ тел дёшево). Но для динозавров/Рима такая же повторяемость окружения быстро **наскучивает**. Исследование искало, как разрешить это напряжение по-разному для разных тем.

## Подход к AI-конвейеру
- Арт карт: image-модели (Nano Banana Pro / GPT Image 2 / Flux).
- Hero-экшены: AI-видео (Kling i2v) → спрайты с альфой/хромакей; используется экономно (несколько карт на колоду).
- 2.5D-параллакс из сгенерированных слоёв.
- Технически-точное (анатомия, орбиты, механизмы, схемы) — **процедурно кодом + вектор**, т.к. image-AI на точных диаграммах ненадёжен.
- TTS-озвучка.
- **БЕЗ «тяжёлого» 3D** (ретопология/риг/анимация под web не амортизируются).
- Реалтайм-3D / world models рассматривались, но отложены как «пока дорого» (это допущение — один из пунктов для ревью).

## Текущее состояние
- Есть рабочий **POC карты T-Rex**: карта → тап, «оживает» AI-видео рёва → квиз из 3 ответов → reward-видео.
- Этот POC заказчик назвал **«скучные кнопки»** — нужны настоящие игровые/интерактивные механики, но конвейерно-дешёвые. Именно это и есть калибратор «как НЕ надо».
- **Пилотная тема = динозавры** (самый сложный случай по переиспользованию; если конвейер вытянет его — остальные темы легче).
- Архитектура колоды, к которой пришли до исследования: 4–5 hero-карт (богатые механики) + ~3 библиотечных механики (масса карт) + сквозной персонаж-гид + коллекционный мета-слой.

## Жёсткие ограничения (контекст для оценки feasibility)
- **Офлайн-PWA** (важно: настоящий AR/WebXR на офлайн-iOS фактически недоступен).
- **Локализация EU + US** (несколько языков — множитель авторских часов).
- **Детская безопасность / IP-чек человеком на каждую карту** (обязательный human-gate).
- **Бюджетная дисциплина**: у провайдеров генерации (fal/Kling/Replicate) нет жёсткого капа расходов — нужен собственный kill-switch.
- Производственная цель — **высокая пропускная способность** («много карт в день»), что и делает баланс вовлечение/стоимость критичным.

## Что исследование должно было дать
1. Библиотеку переиспользуемых интерактивных механик, сбалансированных по вовлечению и стоимости.
2. Конкретные раскладки колод по темам (где высокий reuse, где нужна вариативность).
3. Универсальный фреймворк: «дали новую тему → как собрать колоду».
Всё — как «топливо» для дальнейшего брейншторма и для решения о сроках/стоимости разработки конвейера.
