# 06 — Вертикальный срез: 2 карты на «Interactive AI-Clip Stage + reusable plugins»

> 🔁 **ЗАМЕЩЁН `08-first-pass-plan.md` для выбранного объёма.** Принято решение: **разные колоды, по 1 карте (кросс-тема)** — операционный план в `08` (3 карты: T-Rex / удильщик / сверхновая, тонкий движок). Этот `06` оставлен как **within-deck-альтернатива** (одна колода, несколько карт) — к ней вернёмся, когда будем тестировать внутри-колодную связность и коллекционный слой.

> **Скоуп — по уточнённой границе (`07`):** запрещены ручная анимация, bespoke per-card код и сложные симуляции без reuse; физика/FX **разрешены как reusable config-driven плагины**.
> **Цель среза — де-рискнуть три вещи:** (а) clip-механика даёт настоящую игру лучше «скучных кнопок» POC; (б) **архитектура переиспользуется** — карта №2 другого типа собирается **конфигом, без нового bespoke-кода** (Factory Rule держит); (в) **physics-механика даёт replay/мастерство, которого ветки клипов не дают**.
> **Привязка:** пилот = динозавры; основа — `poc-dino-companion`. POC (`card→tap→roar→quiz→reward`) — анти-эталон.

## 0. Что доказываем

| # | Гипотеза | Как проверяем |
|---|---|---|

## 1. Две карты (одна тема, один Stage, разные механики)

| Карта | `mechanic` / источник движения | Решение (сигнатура) | Pressure | Что нового |
|---|---|---|---|---|
| **№1 «Сбеги от T-Rex»** | `branching-choice` — organic-жизнь = **AI-клип** | замри/беги — зависит от факта «T-Rex наводится на движение» | Quest⇄Challenge | reuse арта+roar POC + 2–3 outcome-клипа (1 Kling-батч) |
| **№2 «Спаси детёнышей»** | `catch-dodge` — **Arcade-physics плагин** (built once) | двигай гнездо, лови падающих детёнышей, уворачивайся от камней | Quest | **только конфиг плагина** + 1 арт + alpha-актёр-луп |

**Контроль:** текущий POC-квиз (`tap→video→3 кнопки`) — мерим разницу.
**Зачем именно такая пара:** №1 — чистый clip; №2 — физика-плагин. Это честно проверяет H1 (Stage + плагин переиспользованы конфигом) и H2 (физика vs клипы по replay).

## 3. Learning contracts

- **№1 «Сбеги»:** понятие = T-Rex наводится на движение, теряет неподвижную добычу; заблуждение = «беги быстрее»; доказательство = осознанно «замер»; near-transfer = другой хищник; later-transfer = «подкрадись».
- **№2 «Спаси»:** понятие = детёныши держатся гнезда / угроза камнепада; заблуждение = «лови всех подряд»; доказательство = приоритет нужных целей; near-transfer = другой вид/угроза; later-transfer = ссылка на №1.

**Тест каждой:** выиграть, **не** поняв факт, — нельзя.

## 5. ⭐ Операционно-безопасный слой — ОБЯЗАТЕЛЕН (учёт §4 ревизии / `05` §7)

## 7. Состав работ и оценка (приор)

**Раз:** Stage runtime (обобщение POC под манифест) + **arcade-physics плагин** + particle/camera-пресеты + линтер + offline + телеметрия + a11y ≈ **~1–1.5 недели** (многоразовое ядро).
**Карта №1 (clip):** reuse ассетов + 2–3 клипа + JSON ≈ 1–2 дня.
**Карта №2 (physics):** конфиг плагина + 1 арт + actor-луп + тюнинг ≈ 1–2 дня, **без bespoke-кода** (если нужен код — H1 под вопросом).

## 8. Критерии успеха / стопа

**Идём на колоду, если:**
- **H1:** карта №2 собрана **конфигом, без нового bespoke-кода** (Stage+плагин переиспользованы) — Factory Rule подтверждён;
- **H2:** physics-карта даёт ощутимо больше replay/мастерства, чем clip-карта (оправдывает плагин-инвестицию);
- **H3/H5:** branch-by-fact учит и переносит; выиграть, не поняв, нельзя;
- обе бьют POC-кнопки; мягкий профиль не хуже для 7–8; срез проходит device-matrix (Arcade-перф + декодеры).

**Стоп / переделка, если:**
- карта №2 потребовала bespoke-кода → плагин/Stage недо-обобщены, доработать config-схему до колоды;
- physics не даёт прироста replay над клипами → пересмотреть, какие механики физические, а какие clip;
- outcome-клипы ветвей «не из одной карты» → закрепить «один input-кадр на все ветви» (`07` §5);
- жёсткий исход растит тревогу младших → Quest дефолт.

## 9. Что НЕ делаем на срезе

**Преждевременно (`05` §6, `07` §4-отложить):** orchestration-стек, админка, multi-provider router, world-models, C2PA-авто, отдельные приложения на колоду, богатый мета-слой, **Matter Slingshot/Stack** (следующим reusable-модулем, не на срезе), TD, fluid/cloth/ragdoll.
**Против границы (`07` §0):** ручная/organic анимация кодом, bespoke per-card код, сложные симуляции без reuse.
