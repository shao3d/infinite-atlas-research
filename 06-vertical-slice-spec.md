# 06 — Спека вертикального среза (4 карты, динозавры)

> **Цель.** Де-рискнуть ядро **до** производства полной колоды. Проверить на реальных детях гипотезы, которые ресёрч (`01`–`04`) и ревизия (`05`) оставили недоказанными — и которые сейчас стоят как 🧪/❓.
> **Почему срез, а не колода.** Вердикт ревизии: не запускать полную линейку и даже полную 16-карточную колоду по текущим гейтам. 4 карты с контролируемыми A/B дают ответ дешевле и честнее.
> **Привязка.** Пилот = динозавры; основа — `poc-dino-companion`. Текущий POC (карта → тап-видео → квиз → reward) служит **анти-эталоном** и калибратором.

---

## 0. Что доказываем (5 гипотез → 5 сравнений)

| # | Гипотеза из `05` | A/B-сравнение |
|---|---|---|
| H1 | Разнообразие даёт **решение/причинная модель**, не жест | **тот же жест / разное решение** |
| H2 | Смена **жеста** при той же задаче слабее меняет опыт, чем смена решения | **разный жест / то же решение** |
| H3 | Жёсткий универсальный провал хуже мягкого обратимого для младших | **мягкий (Quest) / жёсткий (Challenge) провал** |
| H4 | Анимированный **stateful** actor поднимает вовлечение/понимание vs static+shader | **static+shader / stateful animated actor** |
| H5 | Факт-как-правило учит, факт-как-награда — нет («выиграть, не поняв?») | **знание-как-правило / факт-как-награда** |

Если H1/H5 не подтвердятся — рушится вся идея «intrinsic integration», и продукт = весёлая аркада с познавательным шпоном (это надо знать ДО колоды).

## 1. Четыре карты

Все на динозаврах, на **2 engine-family** (creature-arena + dig/assemble) + 1 guide-rig + 1 actor-pack + 1 процедурный hero. Гид колоды — **БИП** (дрон-исследователь, нейтрален, переезжает в другие темы).

| Карта | Движок / family | Сигнатура (решение) | Pressure | Actor-тир | Роль в эксперименте |
|---|---|---|---|---|---|
| **A. «Собери скелет — оживёт по правилу»** | dig/assemble | анатомия определяет исход: верный костяк → встаёт и идёт; неверный → смешной франкен-ковыляющий | Quest | L1 | **H1, H5** (внутри: knowledge-as-rule vs fact-as-reward) |
| **B. «Спаси детёнышей от хищника»** | creature-arena | оттащить/прикрыть детёнышей; рассчитать путь угрозы | Quest⇄Challenge | L1 | **H3** (внутри: мягкий vs жёсткий провал) |
| **C. «Хищник близко: беги ИЛИ подкрадись»** | creature-arena | одно решение «избежать обнаружения», два жеста: swipe-уворот / медленный hold-stealth | Quest | L0⇄L2 | **H2 + H4** (внутри: жест × actor static/stateful) |
| **D. «Метеор: конец эпохи»** | процедурный hero (БЕЗ видео) | прицел/сила/угол → кратер, волна, последствия экосистемы | Challenge | H (процедурный) | контроль hero-пика; пара к A по жесту aim-launch для **H1** |

**Карта-эталон-«как-не-надо»:** оставить рядом доступным текущий POC-квиз (тап→видео→3 кнопки) как пятый, контрольный вариант — чтобы измерить разницу с ним напрямую.

---

## 2. Матрица вариантов (что именно собираем)

Чтобы 4 карты дали 5 сравнений, часть — **внутрикарточные A/B-варианты** (один контент, два билда):

| Сравнение | Реализация |
|---|---|
| H1 тот же жест / разное решение | A (drag-place = собрать анатомию) **vs** D-вариант с drag-place прицела — общий моторный примитив, разные причинные модели |
| H2 разный жест / то же решение | C: «избежать хищника» через **swipe-беги** vs **hold-подкрадись** |
| H3 мягкий / жёсткий | B: **Quest** (нет таймера, детёныш прячется, повтор) vs **Challenge** (таймер, хищник съедает, рекорд) |
| H4 static / stateful actor | C: хищник как **static+shader** (L0) vs **stateful animated** (L2: замечает движение → меняет траекторию, складывает позу перед броском) |
| H5 знание-правило / факт-награда | A: **анатомия определяет** встанет/упадёт **vs** фиксированная анимация + факт озвучен после (награда) |

Итого билдов: A×2, B×2, C×2 (жест) ×2 (actor) = 4, D×1, POC×1 → **~12 сборок** одного контента. Это нормально для эксперимента — мы платим за варианты, не за 12 разных карт.

## 3. Learning contracts (по карте)

Каждая карта несёт контракт (формат для всех будущих карт):

- **A. Скелет:** понятие = осанка/опора определяются скелетом; заблуждение = «динозавр стоит как угодно»; действие-доказательство = верно собранный костяк держит позу; ожидаемая ошибка = ноги/таз не на месте; near-transfer = другой вид (двуногий→четвероногий); later-transfer = на карте C поза хищника перед броском «читается» из той же логики.
- **B. Спасение:** понятие = стратегия хищника (зрение на движение / траектория); заблуждение = «прятаться бесполезно»; действие = увести детёнышей вне линии атаки; near-transfer = другой хищник; later-transfer = C (подкрадись, не вызвав ту же реакцию).
- **C. Беги/подкрадись:** понятие = обнаружение по движению/звуку; заблуждение = «быстрее = безопаснее»; действие = выбрать темп под сенсор хищника; near/later — связан с B.
- **D. Метеор:** понятие = масштаб удара → цепочка последствий (пыль→похолодание→цепи питания); заблуждение = «вымерли мгновенно от удара»; действие = увидеть отложенную причинно-следственную волну.

**Главный тест каждой:** можно ли стабильно выиграть, **не** поняв целевого факта? Если да — карта-кандидат на переделку.

## 4. Что измеряем — на 0 / +1 день / +1 неделя

Не «понравилось», а поведение и перенос (privacy-safe, см. §5):

| Метрика | 0 | +1д | +1нед |
|---|---|---|---|
| Добровольный replay (выбрал переиграть сам) | ✓ | ✓ | ✓ |
| Ранний выход / фрустрация (бросил до конца) | ✓ | | |
| Осмысленные попытки (не случайный тык) | ✓ | | |
| Объяснение причины **своими словами** | ✓ | ✓ | ✓ |
| Near-transfer (принцип на новом объекте) | | ✓ | ✓ |
| Узнавание однообразия («опять то же самое») | ✓ | | ✓ |
| Эмоциональное описание персонажа (для H4) | ✓ | | ✓ |
| Родительская оценка: «обучение или короткая аркада» | | ✓ | |

**Возрастные страты:** 7–8 · 9–10 · 11–12, с разным уровнем чтения/моторики/чувствительности к таймеру. Заранее: критерии остановки и что считаем «значимой разницей» (без претензии на academic power-analysis — это калибровка, не публикация).

## 5. ⭐ Операционно-безопасный слой — ОБЯЗАТЕЛЕН в срезе (учёт §4 / `05` §7)

Не «потом» — встроить с первого билда, иначе срез измеряет не тот продукт:

- **Доступность:** переключатель **reduced-motion**; **таймер-off** (Explore-fallback) для чувствительных; voice-first для слабочитающих 7–8; контраст/цветобезопасность; крупные touch-targets (моторика).
- **Privacy-safe телеметрия:** логировать попытки/подсказки/выходы/replay **локально, без аккаунта ребёнка**, без PII; экспорт обезличенных событий для анализа. Это и есть измерительный инструмент §4.
- **Физ-вход:** сценарии повреждённой карты / плохого света / нет камеры / **ручной ввод кода** / дубликата — карта должна открываться всеми путями.
- **Offline lifecycle (базово):** установка среза как deck-pack, проверка нехватки места, повтор при обрыве, корректное обновление билда (атомарно), без потери прогресса.
- **Device matrix:** минимум один **low-end Android** + один **старый iPhone/iPad** в тесте (перф, память, термал, батарея).
- **Safety/IP:** human-чек арта и фактов каждой из 4 карт (детский продукт) — обязательный гейт, не авто.
- **Этика мета-слоя:** на срезе мета-слой **минимальный и этичный** (видимый прогресс), без редкостей/ежедневок/FOMO — их валидируем отдельно и осознанно.

## 6. Минимальный сквозной пайплайн (13 шагов)

1. **Learning contract** каждой карты (§3).
2. **Card manifest**: сигнатура (§1 `05`), pressure-профиль, provenance, generation-recipe.
3. **Один PWA-runtime** (не build-на-карту).
4. **2 engine-family** на компонуемых примитивах (creature-arena + dig/assemble) — строить **швы**, не всю библиотеку.
5. **Один guide-rig** (БИП) — canonical master → poses → rig; реген запрещён.
6. **Один reusable animated actor pack** (хищник) — atlas по умолчанию; opaque actor + процедурные FX отдельно; для C — варианты L0 и L2.
7. **Одна процедурная hero-композиция** (метеор) — без видео.
8. **Generation recipe** на каждый ассет: модель, версия, refs, prompt, seed, цена, права (для дрейфа §4F).
9. **Ручные approval-гейты:** Safety/IP, факт, арт.
10. **Автотесты:** runtime, ассеты (схема/альфа/safe-area), offline install + rollback, reachability механик.
11. **Print-export + проверка scan/ID** (хотя бы 4 физ-карты/коды — печать может быть черновой).
12. **Child playtest** (страты §4) → **delayed recall/transfer** на +1д/+1нед.
13. **Решение по остальным 12 картам** — только после прохождения.

## 7. Состав работ и оценка (честно, как приор — проверяем)

**Строим:** 2 engine-family (швы + первичные модули поведения) · 1 guide-rig · 1 actor-pack (×2 тира) · 1 процедурный hero · runtime-каркас + offline-базис + телеметрия + accessibility-переключатели · ~12 сборок одного контента.

**Грубая оценка** (по throughput-приору `05` §8, **не факт**): каркас/runtime/offline/телеметрия — отдельный спринт (наибольший риск); 4 карты с вариантами — как ~2 hero + несколько library-эквивалентов ≈ **порядка 3–5 человеко-недель** на сборку до плейтеста малой командой, **плюс** не-параллелящиеся детские тесты и переделки. Цифру **уточняем по факту первого среза** — это и есть один из его результатов.

## 8. Критерии успеха / стоп-критерии

**Идём на полную колоду, если:**
- H1 и H5 подтверждены: смена решения/причинной модели даёт ощущаемое разнообразие, и на «правило»-картах ребёнок объясняет причину и переносит её (а на «награда»-картах — нет);
- мягкий профиль (Quest) **не** уступает жёсткому по добровольному replay у 7–8 и **не** растит фрустрацию/ранний выход;
- stateful actor (H4) даёт прирост понимания/эмоции, оправдывающий L2 (иначе масса остаётся на L0/L1);
- срез проходит device-matrix, offline-install/rollback, и родители читают его как **обучение**, а не аркаду.

**Стоп / переделка, если:**
- выиграть можно, не поняв факт, на большинстве карт → переписать learning-интеграцию, не масштабировать;
- жёсткий провал заметно растит тревогу/отвал у младших → закрепить Quest как дефолт;
- компонуемые слои не дают разнообразия лучше, чем монолит-эквивалент → пересмотреть §2 `05`.

## 9. Что НЕ делаем на срезе (преждевременно — `05` §6)

Большой orchestration-стек · отдельная админка · несколько workflow-движков · Langfuse/MCP на всё · авто multi-provider router · gen-3D · world-model геймплей · отдельные приложения на колоду · полная C2PA-автоматизация · богатый мета-слой/ежедневки. **Но** provenance/лицензии/human-approvals/generation-recipe — с первого дня.
