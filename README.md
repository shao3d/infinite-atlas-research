# Infinite Atlas — актуальный research-пакет

Этот репозиторий — curated-пакет исследований и решений по Infinite Atlas для Степана и внешнего обсуждения.

Это не код приложения и не полный production-репозиторий. Здесь лежат только материалы, которые помогают понять:

- что за продукт строим;
- какие гипотезы уже исследовали;
- какие решения сейчас считаются актуальными;
- где начинается backlog, а где текущая рабочая рамка.

## Короткий путь чтения

1. `INFINITE-ATLAS-BRIEF-FOR-STEPAN-v2a.html`
   Быстрая визуальная сводка решений для Степана.

2. `09-plain-summary.md`
   Простая текстовая сводка: что выяснили и к чему пришли.

3. `INFINITE-ATLAS-CODEX-PROJECT-BIBLE-v1.2.md`
   Актуальная проектная библия: gameplay-first подход, reusable plugins, AI-ассеты, MVP-дисциплина и роль Atlas Shorts.

4. `05-revised-architecture.md` + `07-clip-pipeline.md` + `08-first-pass-plan.md`
   Архитектура, pipeline и первый план реализации как research-контекст.

5. `03-decks/*`
   Раскладки по колодам. Это полезный backlog идей, а не прямое ТЗ на немедленную разработку.

## Главный актуальный статус

- Базовая рамка продукта: **gameplay-first интерактивы + reusable plugins + AI-first ассеты**.
- Atlas Shorts / "ожившие факты" допустимы как optional route, но не являются форматом продукта по умолчанию.
- Текущий MVP должен двигаться быстро: строим playable card first, не строим фабрику заранее.
- Старые документы и числовые оценки из early research считаются гипотезами, пока их не подтвердили дети, production-метрики или ручная проверка.

## Как не запутаться

Если документы спорят друг с другом, приоритет такой:

1. `INFINITE-ATLAS-CODEX-PROJECT-BIBLE-v1.2.md`
2. `05-revised-architecture.md`
3. `07-clip-pipeline.md`
4. `EXTERNAL-REVIEW.md`
5. Старые numbered research docs как backlog и материал для обсуждения

Главное: этот пакет нужен для ясности, а не для усложнения процесса. Для MVP важнее быстро получить работающую playable card, чем заранее построить большую фабрику.
